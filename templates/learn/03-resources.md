# 03-Resources — What This Is

**The knowledge base. Evergreen reference that stays useful over time.**

Resources are things you return to — not things you're working on right now.
A note about JWT auth patterns is a resource. Your notes from a specific
project are not (those live with the project).

---

## The Test

Ask: "Would this still be useful in a year, regardless of what I'm working on?"

If yes → it's a resource.
If it only matters in the context of a current project → keep it in that project.
If you're not sure → Inbox first, decide later.

---

## What Goes Here

- Concepts and frameworks you return to
- Book and article notes
- How-to reference (your own distilled version, not a copy-paste)
- Notes on people you know professionally
- Research you've done that applies beyond one project

---

## What Doesn't Go Here

- Active project notes (01-Projects)
- Ongoing area tracking (02-Areas)
- Things you're currently processing (00-Inbox)

---

## How It's Organized

One folder per topic domain. Start with the defaults and add your own:

```
03-Resources/
├── programming/     ← technical concepts, patterns, architecture notes
├── business/        ← strategy, frameworks, mental models
├── people/          ← one note per person you know professionally
└── reading/         ← book notes, article summaries
```

Don't over-engineer the folder structure. If you have fewer than
10 notes on a topic, a folder isn't worth it yet.

---

## The Knowledge Flow

The value of Resources comes from linking. When you finish a project
and learn something, distill it into a Resource note. Link from future
projects back to it. Over time you build a connected knowledge base
instead of a graveyard of isolated notes.

Your agent can help: "Take what we just figured out and add it to Resources."

---

## People Notes

`people/` deserves special attention. One note per person — their role,
how you know them, what they care about, interaction history.
This isn't a CRM (that's in `07-Systems/CRM/`).
It's the knowledge layer — what you know about them intellectually.
The CRM tracks interactions. People notes capture understanding.
