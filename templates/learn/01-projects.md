# 01-Projects — What This Is

A project is anything with a finish line.

Not a habit. Not an ongoing responsibility. A project has a specific end state — something that will be done and then move to the archive. "Get healthy" is not a project. "Run a 5K on March 15th" is a project.

## The test

Ask: "Will this ever be done?" If yes, it's a project. If it's something you'll always be doing — maintaining your health, staying on top of finances — it lives in `02-Areas/` instead.

## What a project contains

Each project gets its own folder:
```
01-Projects/project-name/
├── README.md     ← what it is, why it matters, when it's done
├── notes.md      ← working notes, research, thinking
└── assets/       ← files, images, attachments
```

The `README.md` is the most important file. It answers three questions:
- **What is this?** One sentence.
- **Why does it matter?** Which role in your life does it serve?
- **Done when?** The specific, observable end state.

If you can't answer "done when," it's not a project yet.

## How many projects

Most people do well with 5–15 active projects. More than that and nothing gets real attention. Your agent will flag if you're accumulating too many.

## When a project ends

Move the whole folder to `04-Archive/projects/`. Never delete — archived projects are a record of what you've done and often contain useful reference material.
